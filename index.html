Offline-first • Encrypted • Quick daily UX

Secure secrets — designed like an Android app, documented like a real project.

NEOKey is an offline-first Android authenticator for storing secrets and generating strong passwords. Sensitive data is encrypted at rest and unlocking supports biometrics with a master PIN fallback. This page documents the concept, UI flow, tech decisions, security considerations, and testing outcomes.

Security Notes

Document your security decisions and what issues you found + fixed. Keep it honest and practical (what you protected against, and what is out of scope).

What we protect
  • Secrets should not be stored in plaintext on disk.
  • App should require authentication before revealing/copying secrets.
  • Back navigation should not expose invalid states or bypass locks.
Implementation checklist (adapt later)
  • Encrypt before writing to Room; decrypt only when needed.
  • Use Keystore-backed key material (avoid hardcoding secrets).
  • Biometric prompt with fallback (PIN/password).
  • Mask secrets by default; explicit “reveal” action.
  • Defensive navigation (handle spam/back/edge cases).
